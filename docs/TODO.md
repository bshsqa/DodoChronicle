## 우선순위 작업 목록

- [ ] **#36 Gemini API 키 입력 후 모델 목록 로딩 실패 수정**
  > 옵션에서 Gemini API 키를 입력한 뒤 모델 목록이 로딩되지 않는 문제를 수정한다. 기존 키 유지 흐름, API 실패 메시지, 모델 목록 응답 파싱 로그를 함께 확인한다. ENH-020 범위.

- [ ] **#38 초기 선택 대표 사진이 옵션 상단에 표시되지 않는 문제**
  > 초기 세팅에서 선택한 아이 사진이 옵션 다이얼로그 상단 대표 사진 영역에 표시되지 않는 문제를 수정한다. `Child.referencePhotoUri` 저장, state 복원, URI 권한/이미지 로딩 실패 fallback을 확인한다. ENH-020 범위.

- [ ] **#37 사진 이벤트 날짜 결정 로직 개선**
  > `DATE_TAKEN=0` 등으로 사진이 `1970-01-01`에 배치되는 문제를 해결한다. 전체 갤러리 row를 한 번 훑고, 각 사진은 얼굴 분석 전에 `DATE_TAKEN → EXIF → DATE_ADDED → DATE_MODIFIED → file lastModified` 순으로 날짜만 먼저 확정한다. 초기 분류에서는 확정 날짜가 `birthDate - 1년` 이후인 사진만 scan item으로 저장한다. ENH-021 범위.

- [ ] **#39 초기 사진 분류를 생년월일 기준 백그라운드 작업으로 전환**
  > 초기 세팅에서 전체 분석 완료를 기다리지 않고 앱에 진입하게 한다. 전체 갤러리 row를 날짜만 먼저 판정하고, 아이 생년월일 1년 전 이후 사진만 백그라운드 얼굴 분석 대상으로 삼는다. 상단 배너로 진행률/완료 상태를 표시하고 중단 후 재시작 시 이어서 진행한다. ENH-021 범위.

- [ ] **#40 대량 사진 그룹 선택 UX 개선**
  > 분류 완료 후 그룹 선택창에서 `추가 / 닫기 / 완료` 흐름을 지원한다. `추가`는 선택 그룹만 이벤트로 등록하고 목록에서 제거, `닫기`는 캐시 유지, `완료`는 확인 팝업 후 남은 그룹 캐시와 배너를 제거한다. ENH-021 범위.

- [ ] **#30 Gemini API 키 입력**

- [ ] **#34 문맥 검색 결과 업그레이드**
