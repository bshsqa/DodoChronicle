# ENH-002: 동적 얼굴 모델 갱신 및 타임라인 UX/UI 개편

## 1. 개요
앱의 메인 기능인 얼굴 인식 구조와 타임라인 표시 UI를 전면 개편합니다.
초기화 시 5,000장 스캔의 중복 연산을 제거하고, 아이의 성장(얼굴 변화)에 모델이 적응할 수 있도록 기준 임베딩을 동적으로 갱신합니다. 또한 타임라인을 날짜별로 그룹화하여 가독성을 높이고, 다중 선택 모드를 도입하여 '학습 제외' 및 '삭제' 관리를 직관적으로 개선합니다.

## 2. 요구사항

### 2.1. 얼굴 모델 동적 갱신 및 등록 구조
1. **초기화 시 일괄 등록**: 초기화 마법사에서 클러스터 선택 완료 시, 사용자가 선택한 클러스터에 속한 모든 사진을 타임라인 DB에 "확실한 아이 사진"으로 즉시 자동 일괄 등록합니다. (추가 스캔/판별 생략)
2. **동적 임베딩 갱신 (Rolling Window)**: 타임라인에 사진이 추가/삭제되거나 상태가 변경될 때마다, 모델 오염에서 제외되지 않은 최신 50장(촬영일 기준)의 임베딩 평균을 재계산하여 `Child`의 기준 벡터를 실시간 갱신합니다.

### 2.2. 타임라인 날짜별 UI 그룹핑
1. **날짜별 상자(Card) 표시**: 기존의 이벤트 개별 렌더링 방식에서 탈피하여, 같은 날짜(`LocalDate`)의 사진과 텍스트 이벤트들을 하나의 "일간 카드(Daily Event Card)"로 묶어서 표시합니다.
2. **상세 뷰(Detail Screen/Dialog) 제공**: 일간 카드를 클릭하면 해당 날짜의 상세 뷰가 열립니다. 상단에는 사진 갤러리가, 하단에는 텍스트(한 일, 한 말 등) 이벤트 목록이 표시되어 가독성을 높입니다.

### 2.3. 상세 뷰 다중 선택 모드 및 학습 제외 기능
1. **다중 선택 모드 활성화 (Multi-select)**: 상세 뷰 상단의 사진 영역에서 특정 사진을 꾹 누르면(Long Press) 여러 사진을 고를 수 있는 '다중 선택 모드'로 진입합니다.
2. **상단 액션 바 (Contextual Action Bar)**: 선택 모드 시 상단 타이틀 바에 [삭제] 아이콘과 [학습 제외/포함] 토글 아이콘이 노출됩니다.
3. **혼합 상태(Mixed State) 방어 로직**: 사용자가 '학습 중'인 사진과 '학습 제외' 상태인 사진을 섞어서 선택한 경우, [학습 제외/포함] 버튼은 비활성화(Disabled)되며 오직 [삭제] 버튼만 동작합니다.
4. **제외 마커 (Exclude Indicator)**: 학습에서 제외된 사진은 상세 뷰 갤러리의 각 사진 우측 상단에 작은 제외 마커(예: 빗금 친 눈 아이콘)를 오버레이로 띄워 시각적으로 식별할 수 있게 합니다.

## 3. 구현 계획

### 3.1. Data Layer 변경
- `PhotoRecordEntity`, `PhotoRecord`에 `isExcludedFromModel: Boolean` 필드 추가 및 Room DB 마이그레이션.
- 최신 50개 임베딩 획득을 위한 DAO 쿼리 추가.

### 3.2. Domain Layer 추가
- `UpdateChildEmbeddingUseCase` 추가: DB에서 최신 50개 임베딩을 모아 평균 계산 후 `ChildRepository` 업데이트.

### 3.3. Presentation Layer 변경
- `TimelineScreen`: 이벤트 목록을 `events.groupBy { it.date }`로 그룹핑하여 `DailyEventCard` 형태의 UI로 렌더링.
- `DailyDetailDialog` (또는 Screen): 그룹화된 하루치 데이터를 받아와 상단 사진 Grid, 하단 텍스트 Column으로 분리 렌더링.
- **다중 선택 상태 관리**: `var selectedPhotoIds by remember { mutableStateOf(setOf<String>()) }` 추가.
- **혼합 상태 판별 로직**: `selectedPhotoIds`에 해당하는 사진들의 `isExcludedFromModel` 값이 모두 같은지 체크하여 상단 액션 바 토글 버튼의 `enabled` 상태 결정.
